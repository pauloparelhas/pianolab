## CONTEXTO DO PROJETO

Estamos construindo um curso interativo de piano chamado PianoLab. Público-alvo:
criança de 7–9 anos, usando celular ou tablet com supervisão do pai.

Stack atual (Fase 1 — sem build step):
- HTML + CSS + JS puro
- Web Audio API (áudio sintético, triangular — ainda sem samples reais)
- Fullscreen API, SpeechSynthesis API (TTS)
- localStorage para progresso
- Publicado no GitHub Pages

Arquitetura atual:
- base.js     → motor: áudio, teclado piano, teoria musical, nav, TTS, LockMode
- piano.css   → todo visual do teclado (ponto único de controle)
- base.css    → design system: cores, tipografia, layout
- m1.html     → Módulo 1: O Dó Central (concluído)
- m2.html     → Módulo 2: As 5 Primeiras Notas (concluído, com layout novo)
- index.html  → Hub de navegação (precisa de redesign)

Currículo planejado:
1. Notas (nome + posição no teclado) ← estamos aqui
2. Escalas (maior, menor, todas as 12 tonalidades)
3. Intervalos
4. Acordes e cifras
5. Campo harmônico
6. Progressões e padrões
7. Transposição
8. Frases musicais e leitura de partitura

Layout atual dos módulos (M2 em diante):
- .piano-body: corpo escuro inspirado no Yamaha P-125
- .digital-panel: painel LCD/OLED com navegação e conteúdo
- .piano-stage: teclado ocupa destaque, preenche 100% da largura
- Portrait: painel cima + teclado baixo | Landscape: painel esquerda + teclado direita
